Android相关文件结构

1. APK的组成

  1. AndroidManifest.xml: Android的基础配置文件，包含了与apk相关的大量信息

  2. asserts:资源文件目录，在java中用AssetManager访问

  3. res：资源文件目录，在java中使用R.xx来使用

  4. lib：so库存放目录，一般由NDK编译得到

  5. META-INF：工程属性文件存放目录

  6. classes.dex：java代码编译得到的Dalvik VM能直接执行的文件

  7. resources.arsc：res目录资源索引文件，保存了原工程中 strings.xml等文件内容

2. 核心基础

Dalvik字节码

Dalvik 是 google 专门为 Android 操作系统设计的一个虚拟机，经过深度优化。虽然 Android 上的程序是使用 java 来开发的，但是 Dalvik 和标准的 java 虚拟机 JVM 存在不同之处 | | JVM | Dalvik | | ——– | ———- | ————————- | | 原理 | 基于栈 | 基于寄存器 | | 执行格式 | java字节码 | dex （dalvik executable） | | 速度 | 普通 | 更快 | | 空间占用 | 普通 | 更少 |

  1. 通过 Dalvik 的字节码我们不能直接看到原来的逻辑代码，这时需要借助如 Apktool 等工具来查看。最终修改 APK 需要操作的文件是 .smali 文件，而不是导出来的 Java 文件重新编译。
  2. Dalvik字节码有两种类型：原始类型；引用类型（包括对象和数组）
  3. 在Davlik字节码中，寄存器都是32位的，能够支持任何类型，64位类型（Long/Double）用2个寄存器表示

Smali

  1. Smali，Baksmali 分别是指安卓系统里的 Java 虚拟机（Dalvik）所使用的一种 dex 格式文件的汇编器，反汇编器。它实现了 .dex 格式所有功能

  2. 当我们对 APK 使用Apktool等进行反编译后，生成Smali相关文件。

3. 深入理解Smali

1.Dalvik字节码原始类型

B—byte 字节
C—char 字符
D—double 双精度浮点数
F—float 单精度浮点数
I—int 整型
J—long 长整型
S—short 短整型
V—void 无
Z—boolean 布尔型
[XXX—array [，表示为数组，后接基本类型
Lxxx/yyy—object L,表示方法，，格式 Ljava/lang/String;（分号结尾） ，String表对象
Lxxx/yyy—$object 表内部类，关键符号$

2.方法的定义

Func-Name (Para-Type1Para-Type2Para-Type3...)Return-Type

举个栗子
hello (Z[I[ILjava/lang/String;J)Ljava/lang/String; String hello (boolean, int[], int[], String, long)
hello (III)Z boolean hello(int, int, int)

3.Smali基本语法
.method 方法
.parameter 方法参数
.prologue 方法开始
.line 123 此方法位于第123行
invoke-super 调用父函数
.field private isFlag:z 定义变量
const/high16 v0, 0x7fo3 把0x7fo3赋值给v0
invoke-direct 调用函数
return-void 函数返回void
.end method 函数结束
new-instance 创建实例
iput-object 对象赋值
iget-object 调用对象
invoke-static 调用静态函数

与跳转有关的分支结构
“if-eq vA, vB, :cond_**” 如果vA等于vB则跳转到:cond_**
“if-ne vA, vB, :cond_**” 如果vA不等于vB则跳转到:cond_**
“if-lt vA, vB, :cond_**” 如果vA小于vB则跳转到:cond_**
“if-ge vA, vB, :cond_**” 如果vA大于等于vB则跳转到:cond_**
“if-gt vA, vB, :cond_**” 如果vA大于vB则跳转到:cond_**
“if-le vA, vB, :cond_**” 如果vA小于等于vB则跳转到:cond_**
“if-eqz vA, :cond_**” 如果vA等于0则跳转到:cond_**
“if-nez vA, :cond_**” 如果vA不等于0则跳转到:cond_**
“if-ltz vA, :cond_**” 如果vA小于0则跳转到:cond_**
“if-gez vA, :cond_**” 如果vA大于等于0则跳转到:cond_**
“if-gtz vA, :cond_**” 如果vA大于0则跳转到:cond_**

4.Smali文件的信息

  1. .class public Lcom/aaaaa; （它是com.aaaaa这个package下的一个类）
  2. .super Lcom/bbbbb; （继承自com.bbbbb这个类）
  3. .source “ccccc.java” （一个由ccccc.java编译得到的smali文件）

5.Smali的声明

# annotations 
.annotation system Ldalvik/annotation/MemberClasses; 
value = { 
      Lcom/aaa$qqq;, 
      Lcom/aaa$www; 
      }
.end annotation

6.Smali的成员变量

  1. 格式

    .field public/private [static] [final] varName:<类型>

  2. 相关处理命令

    1. 获取：iget、sget、iget-boolean、sget-boolean、iget-object、sget-object
    2. 操作：iput、sput、iput-boolean、sput-boolean、iput-object、sput-object

  3. 指令实例

    sget-object v0, Lcom/aaa;->ID:Ljava/lang/String;
//获取ID这个String类型的成员变 量并放到v0这个寄存器中

    iget-object v0, p0, Lcom/aaa;->view:Lcom/aaa/view;
//iget-object指令比sget-object多了一个参数，就是该变量所在类的实例，在这里就是p0即“this”

    const/4 v3, 0x0 sput-object v3, Lcom/aaa;->timer:Lcom/aaa/timer;
//timer=null

7.Smali函数分析

  1. 函数类型

    direct virtual
    private函数 其他函数
    invoke-direct invoke-virtual

  2. .invoke-static：用于调用static函数

    invoke-static {}, Lcom/aaa;->CheckSignature()Z
//{}，其实是调用该方法的实例+参数列表

  3. .invoke-virtual

    sget-object v0, Lcom/dddd;->bbb:Lcom/ccc; 
invoke-virtual {v0, v1}, Lcom/ccc;->Messages(Ljava/lang/Object;)V
//v0是bbb:Lcom/ccc
//v1是传递给Messages方法的Ljava/lang/Object参数

  4. .invoke-direct

    invoke-direct {p0}, Landroid/app/TabActivity;->()V
//init()就是定义在TabActivity中的一个private函数

  5. .invoke-super

  6. 函数结果返回

    const-string v0, "Eric" 
invoke-static {v0}, Lcmb/pbi;->t(Ljava/lang/String;)Ljava/lang/String; 
move-result-object v2
//调用的函数返回非void，那么还需要用到move-result（返回基本数据类型）和move-result-object（返 回对象）指令
//v2保存的是调用t方法返回的String字符串
