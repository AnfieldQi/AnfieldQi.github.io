Hugo+Github Pages

Written by anfieldqi with ♥ on April 12, 2020 in 教程工具 , 1354 words

Hugo简介

Hugo是由Steve Francis基于Go语言开发的静态网站构建工具。

Hugo博客搭建流程

准备工作
  1. github账号：使用github pages 来搭建,网址就不写了
  2. git:用来将博客发布到github上，众所周知
  3. hugo：一个博客框架，将Markdown渲染为静态网页
环境部署
hugo

  1. 下载Hugo，根据自己的系统选择不同的版本，本文所用环境为windows

  2. 选择合适的目录解压下载的文件，包含hugo.exe等三项内容

  3. 配置变量

    3.1 在目录下建立名为bin的文件夹，将内容移动到此处

    3.2 在系统变量设置中将路径（如：D:/hugo/bin）加入Path

    3.3 在命令行输入hugo version，输出版本号，安装成功

git

  1. 下载git安装包，根据系统选择

  2. 安装后在命令行中输入git version,输出版本号则表示安装成功

  3. 后续还需要配置密钥信息，这个自行查找相关教程就好

github
  1. 登录github,新建一个名为<账户名>.github.io的库，注意不要带README
  2. 选择此库的 Settings ,拖动至github pages块，并点击 Automatic page generator
  3. 然后github会自动为我们创建，稍等一下，https://«账户名>.github.io就可以访问了
hugo使用
建立新站点

1.在选好的本地路径下，使用：

hugo new site Blog

此时会创建一个一个名为Blog的文件夹来存放博客

2.进入文件，目录结构如下：

├── archetypes # 创建新md页面时的预设信息存放路径
├── content # 写作的内容都存放在这里面
├── data
├── layouts # 网站模版
├── static # 包含 CSS、JavaScript、Fonts、media 等，与网站页面设置有关
├── themes # 主题
└── config.toml # 配置文件
选择喜欢的主题

1.在hugo官方主题页选择自己喜欢的主题

2.将选择的主题下载后放入themes文件下

创建页面
hugo new readme.md

使用此命令后在content页面下会出现readme.md文件

1.打开文件后可以看见title等信息，为yaml语法，这里的默认项可以在archetypes目录相关文件中修改

  • title:
  • date:
  • draft: # 是否为草稿，发布时需修改为false
项目build
  1. 生成网站
hugo server --buildDrafts -w

此时可以在本地1313端口访问网站

  1. 此时仅是本地预览阶段，-w代表每次修改可以直接在启动的页面中显示,若要发布，将所写页面中draft修改为false

  2. 运行hugo命令,此时在博客目录下生成一个public文件

    hugo

  3. 进入public文件，初始化git，将此部分push至GitHub相关库中。

  4. 接下来，就可以在https://<用户名>.github.io下访问自己的博客了
