项目部署至服务器

1.配置服务端环境

1.1 python环境

一般现在服务器自带2和3环境，根据需要使用即可

1.2 创建虚拟环境

为了方便项目管理，防止环境依赖冲突，建议部署时使用虚拟环境

  1. 下载 **virtualenv **库

    pip3 install virtualenv

  2. 下载**virtualenvwrapper-win **库

    pip3 install virtualenvwrapper-win

  3. vim ~/.bashrc配置环境变量

    export WORK_HOME=$HOME/.virtualenvssource /usr/local/python3/bin/virtualenvwrapper.sh
source ~/.bashrc

  4. 定位至虚拟环境放置路径

    #创建了一个名为env_virtual的虚拟环境
#python指定版本，系统默认为python3，如果系统默认为2.7，使用--python=python3版本
mkvirtualenv env_virtual --python=python

  5. 在虚拟环境中安装项目相关依赖库，跟平时安装一样

2 远程连接

linux：ssh

windows：搞个xshell

3 运行程序

命令行的典型交互形式为session，其特点是：窗口与其启动的进程是关联的。打开窗口，会话开始；关闭窗口，会话结束，会话内部的进程也会随之终止。所带来的问题是，如果典型应用场景使用ssh，那么ssh会话关闭，在此会话中的进程也将被关闭，但我们所期望的是服务持续运行

3.1 nohup命令

nohup python -u python_test.py > test.log 2>&1 &

如果程序本身没有问题，则此后程序会在后台运行,输出会重定向到test.log里，使用下面的命令查看python相关进程：

ps -ef | grep python

如果需要停止程序，则杀死对应进程，对应PID使用上一条命令得到

kill PID

3.2 Tmux

将会话与窗口的"解绑"的工具，将它们彻底分离，不会出现窗口关闭会话消失进程停止的情况

作用

  1. 允许在单个窗口中，同时访问多个会话。这对于同时运行多个命令行程序很有用。
  2. 允许新窗口"接入"已经存在的会话
  3. 允许每个会话有多个连接窗口，可以多人实时共享会话
  4. 支持窗口任意的垂直和水平拆分

用法(一般操作流程)

  1. 下载：sudo apt-get install tmux
  2. 启动：terminal中输入tmux
  3. 新建指定名称的会话：tmux new -s
  4. 在里面一系列操作
  5. 分离会话：按下Ctrl+b d或者输入tmux detach命令
  6. 分离后关闭窗口也不会影响进程了
  7. 查看当前所有会话：tmux ls
  8. 接入某会话：tmux attach -t
  9. 杀死会话：tmux kill-session -t
  10. 切换会话：tmux switch
  11. 重命名会话：tmux rename-session -t
  12. 还涉及到一些窗口拆分的东西，暂时没用上，可以查阅相关文档进行使用
