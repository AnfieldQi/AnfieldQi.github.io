IoT固件提取与解包

Written by anfieldqi with ♥ on April 28, 2020 in IoT安全 , 2015 words

1.IoT Firmware

1.IoT设备

IoT 设备是一类特殊的嵌入式设备，基于智能单元与网络提升了传统硬件功能的设备

IoT Device 
= Embedded Device + Network
= Hardware Device + Computation Intelligence + Network

2.IoT固件

IoT Firmware是最终存储运行在微处理器内存中的一段可执行的软件程序

2.1 固件的存储

1.集成式固件存储

  1. 适用：设备功能单一，不需要运行复杂的系统或者通过复杂的协议，也不需要什么计算高级计算功能，厂商会把固件集成到他们的主控芯片中，以节约成本。

  2. 主控芯片MCU：CPU+外接口+内存+flash~~(一般用来控制外围传感器)~~

    如同计算机一样，固件一定被存储于ROM+Flash~~(硬盘+内存)~~中。

  3. 提取方法，只要知道产品所利用的主控芯片地址结构分配，就可以通过某种方法提取

2.分离式固件存储

  1. 适用：比较大一些的主控，将存储器外接

  2. 提取方法：

    2.1 将外接的存储器直接取下来读取，利用相应读取设备，插入电脑，利用配套软件即可读出固件

    2.2 通过Bootloader或者利用JTAG/SWD等调试接口，通过控制主控器的读写存储器流程，进而读出固件内容

2.2 固件的获取方法

1. 固件升级流程截取

  1. 协议：FTP，HTTP

  2. 场景：一般适用于设备初次使用时，会自动进入固件升级流程。可以通过burp,wireshark等抓取

  3. 特点：不需要控制设备

  4. 修复手段：使用加密传输

2. 直接读取存储器

  1. 器材：编程器+焊接器+配套读取编程器软件

  2. 场景：分离式固件存储方式，拆掉存储器，读取固件

  3. 使用：使用编程器读取芯片，使用配套软件读取编程器中的信息，拿到固件文件信息

  4. 特点：不需要控制CPU

  5. 修复手段：加密固件。

    例子：如加密除Bootloader以外的代码，解密程序在Bootloader中实现。解密密钥相关的信息完全可以放到主控器中，并加以读保护，这样，即便他们拿到了加密的固件，和Bootloader中的算法，也没办法得到解密密钥，获取不到有用的信息（来源于绿盟科技博客）

3. 通过通信总线读取（串口）

  1. 器材：通信线+UART接口

  2. 场景：较为方便的读取手段

  3. 使用：通过主板上暴露的UART接口，在PC与固化在主控器中的Bootloader程序之间建立通信

  4. 特点：控制主控中的Bootloader来读取存储器内容

  5. 修复手段：设置主芯片的读写保护

  6. 原理：

    BootLoder存储在 STM32 器件的内部ROM 存储器（系统存储器）中。在生产期间由ST编程。其主要任务是通过一种可用的串行外设（USART、CAN、USB、I2C 等）将应用程序下 载到内部Flash 中。每种串行接口都定义了相应的通信协议，其中包含兼容的命令集和序列。”

    其他信息参考bootloader相关文档

4. 通过调试接口读取

  1. 器材：硬件调试器
  2. 场景：相对于串口可以设置断点等，方便开发者调试
  3. 使用：利用相关调试接口和相配套的硬件调试器及其配套软件，将PC和物联网设备建立连接，使用PC运行调试器配套软件，即可发送命令。不同调试器配套软件不同。
  4. 特点：控制整个设备，开发者调试模式
  5. 修复：隐藏接口，让攻击者难以找到对应针脚

5.串口识别（对应3）

为了找出 UART 的接口，可以使用万用表来确定每个引脚 (TXD，RXD，GND 和 VCC)，下面列出关键的步骤 :

  1. 将万用表功能开关定位到二极管 ( 蜂鸣 ) 档位，关闭设备电源，用黑色探头接到硬件电路板的接地引脚 ( 这

个引脚一般于较大的铜箔面相连 )。

  1. 用红色探头分别放在可能是 UART 接口的每个焊盘或者针脚上，直到听到蜂鸣器 (BEEP) 的声音。

  2. 听到蜂鸣器声音时即可判定设备接口的接地引脚，这种测试方法也称为通断测试。

  3. 将万用表功能开关定位 20V 直流电压档位置，黑色探头保持接口的接地引脚 (GND)，用红色探头移到接口的其他引脚（GND 除外）上。接通设备电源，如果看到恒定电压 (3.3V 或者 5V) 的引脚就是 VCC 引脚。

  4. 再次重启设备并测量剩余焊盘和 GND 之间的电压（除了前面步骤中确定的 VCC 和 GND）。由于在启动期间最初进行的大量数据传输，可以看到在最初的 10-15 秒内其中一个引脚上的电压值发生了巨大波动。这个引脚将是 TXD 引脚。

  5. RXD 可以通过在整个过程中具有最低电压波动和最低总值的引脚来确定。通过以上方法确定的引脚比较集中在某个区域时即可判定这个接口为设备的 UART 接口

6.USB-TTL(对应3 4)

该设备一端接入 PC 机的 USB 接口，另一端有 TXD、RXD、GND、5V、3.3V 五个引脚，只需要将对应的引

脚分别与电路板的 RXD、TXD、GND 引脚相连 (Rx 和 Tx 交叉相连，VCC 可以不连 ) 即可。同时在 PC 机上安装

对应的 USB 转串口驱动，就能让 PC 机识别该串口设备。

2.3. 固件解包

1.binwalk

#常用的命令

#固件信息扫描
binwalk  firmware.bin

#提取文件
binwalk -e firmware.bin

#根据magic签名扫描结果递归提取固件
binwalk -Me firmware.bin

#文件比较
binwalk -W firmware.bin firmware2.bin firmware3. bin

#读取日志
binwalk -f firmwar.log -q firmware.bin

#进行熵分析生成图
binwalk -E firmware.bin

2.iv.foremost

在数字取证和CTF中常用来恢复、分离文件。它默认支持19种类型文件(jpg, gif, png, bmp, avi, exe, mpg, mp4, wav, riff, wmv, mov, pdf, ole, doc, zip, rar, html, cpp 等文件)的扫描识别恢复。
