单元测试和文档测试

1.单元测试

单元测试的意义

单元测试在未来项目重构中有十分重要的作用，单元测试包含了对某功能测试的一切边际条件，未来重构时，只要再运行单元测试，就可以测试重构部分是否对原功能产生影响

没啥可写的，记几个点吧

name context
unittest 单元测试模块
unittest.TestCase 测试类的父类
test_func 测试方法以test开头
assertEqual(d.key, ‘value’) 是否相等
assertTrue(‘key’ in d) 是否为真
with self.assertRaises(KeyError): value = d[‘empty’] 是否抛出指定类型的异常
setUp() 测试类运行前运行
tearDown() 测试类运行后运行

运行：unittest.main()
