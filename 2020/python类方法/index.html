普通方法

实例方法

  1. 所属：实例对象

  2. 调用：用实例化对象调用

类中最常用的方法是实例方法, 即通过通过实例作为第一个参数的方法

class Kls(object):
    def __init__(self, data):
        self.data = data
    def printd(self):
        print(self.data)
test1 = Kls('test1')
test2 = Kls('test2')
test1.printd()
test2.printd()

其中self指向实例方法。通常我们这样实例化对象并使用实例对象的方法。

类方法

  1. 所属：类

  2. 主要用途：作为构造函数，class method可以用来为一个类创建一些预处理的实例.

    有些方法不需要在实例中运行，只在类中运行就可以了。

  3. 调用：调用时将Num类作为对象隐式地传入方法，使用实例化对象和类名调用

  4. 普通做法：

  5. 使用@classmethod修饰

    class Kls(object):
    no_inst = 0
    def __init__(self):
        Kls.no_inst = Kls.no_inst + 1
    @classmethod
    def get_no_of_instance(cls_obj):
        return cls_obj.no_inst
ik1 = Kls()

print ik1.get_no_of_instance()
print Kls.get_no_of_instance()

    无论是从实例调用还是从类调用，它都用第一个参数把当前类对象传递过来，访问类相关参数

  6. 类方法对静态方法的调用

    class Pizza(object):
    def __init__(self, radius, height):
        self.radius = radius
        self.height = height
   
    @staticmethod
    def compute_area(radius):
        return math.pi * (radius ** 2)
   
    @classmethod
    def compute_volume(cls, height, radius):
        return height * cls.compute_area(radius)
   
    def get_volume(self):
        return self.compute_volume(self.height, self.radius)

静态方法

  1. 所属：似乎属于类，但与类和实例都无关，是一些跟某一指定类有关的东西，可以放在类外面，但是会造成混乱

  2. 主要用途：限定namespace，也就是说这个函数虽然是个普通的function，但是它只有这个class会用到，不适合作为module level的function。这时候就把它作为staticmethod

  3. 调用：能用类名和实例化对象调用

  4. 规则：Python不会为Pizza对象实例化静态方法

  5. 比如：

    IND = 'ON'
def checkind():
    return (IND == 'ON')
class Kls(object):
     def __init__(self,data):

  6. 使用@staticmethod修饰

    IND = 'ON'
class Kls(object):
    def __init__(self, data):
        self.data = data
    @staticmethod
    def checkind():
        return (IND == 'ON')
    def do_reset(self):
        if self.checkind():
            print('Reset done for:', self.data)
    def set_db(self):
        #其他方法可以省去实例化等东西就可以使用它
        if self.checkind():
            self.db = 'New db connection'
        print('DB connection made for: ', self.data)
ik1 = Kls(12)
ik1.do_reset()
ik1.set_db()

  2. 题外话：类变量和实例变量的访问

    1.类变量访问：在类内部和外部都用类名.类变量
  在类内部，也可以用 self.类变量-->含义变成一个实例变量（实例变量会屏蔽掉类变量的值）
2.实例变量：在类内部，用 self.实例变量 访问|在类外部，用 实例名.实例变量 访问（实例变量是绑定到一个实例上的变量，它只属于这个实例。这一点区别于类变量，类变量是所有来自于同一个类的实例所共享的）
3.总结
3.1、类变量可以用 类名.类变量 和 self.类变量 两种方式访问，后者一般情况不建议使用。
3.2、类变量是所有对象所共享的，无论任何时候都建议用类名的方式访问类变量。
3.3、实例变量在类内部用 self 访问，在类外部用实例名访问。
3.4、类变量通过 self 访问时则被转化为实例变量，被绑定到特定的实例上，其值会屏蔽掉类变量。
3.5、通过对实例变量（self）的形式对类变量重新赋值后，类变量的值不随之变化。
3.6、方法内的局部变量会屏蔽掉类变量和实例变量。
3.7、同一实例变量在不同的实例中可能拥有不同的值。

staticmethod和classmethod

  1. 两者的区别在于在存在类的继承的情况下对多态的支持不同
  2. Python中不管是classmethod还是staticmethod都有OOP、多态上的意义
